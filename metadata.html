In today's world, having the right kind of knowledge is so important, but more so data, which is the root of so much of what we call knowledge. Data is one of the most valuable assets an individual, a business, nation and the world at large, can have. It is the bedrock of our basic needs, it helps in making informed decisions, it enables key strategic initiatives and with the rise of software, the utilization of data is immense. As its demand increases, so does data. That’s why it’s vital to utilize the right tools and technologies to fully leverage all available data and make it as accurate as possible. Low-quality data adversely impacts performance in the area of need which is why ensuring data quality should be a top priority for all.

What is Metadata

Good data quality starts with metadata. Metadata answers the who, what, when, where, why, and how questions for users of the data. It characterizes data and provides documentation such that the data can be understood and more readily consumed by users. Metadata is required to place the data into proper categories for determining the controls that are needed to assure quality and to determine which regulations apply to the data.

The goal of this project

To provide more information about a given file.

- To provide more information about a given file.
- To store uploaded files for users so that they can come back to them at any given time.
- To allow users to share or download their metadata for other ethical purposes.
- To give users an appealing application that is comfortable to use.

The project scope

- Upload a file and get the available metadata
- Create an account.
- Login.
- Share metadata.
- Download metadata.
- Give an app review.
- Save the metadata along with the file on a database for later use.

Target Audience

- Photographers who need the details from the earlier saved or prepared document to support their profession.
- Security organizations that require access to vital information in order to protect the nation, particularly from cybercrime.

Features in the web application

-Login: Users are required to create an account and to login before they use the system so as to make sure we know who is uploading what to prevent misuse of the system for illegal purposes.
- File Upload: Users can upload files in the format of pdf, txt, csv, json, jpg, jpeg.
- Login.
-Generate Metadata: Metadata of the uploaded file is immediately provided to the user once it is uploaded without having the user to click on any other button or link to have good user experience.
- Account Settings: Users can view, edit their passwords.
- Manage Uploaded Files: Users can view and save files and/or metadata of the files they have uploaded before so that they can use the information again.

System Limitation

The application is only available for use on the web. There is no native app made for different operating systems for now. Editing the metadata is not possible.
